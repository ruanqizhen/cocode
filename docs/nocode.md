# 零基础启航：非程序员的 AI 极速编程指南

> 自然语言是世界上最新、最强大的编程语言。任何人，只要能清晰地表达自己的想法，就能让 AI 替你写出可运行的程序。



在过去，写程序是极少数人的专属技能。你需要学习晦涩的语法，配置复杂的开发环境，还要和各种奇怪的报错死磕。

但今天，大语言模型（LLM）彻底打破了这堵高墙。哪怕你没有任何编程背景，一辈子没写过一行代码，也完全可以利用通用的 AI 对话工具（如 ChatGPT、Claude、Kimi 等），在几分钟内做出一个完全属于你自己的小工具或网页。

本章将为你揭开这个秘密：非程序员该如何用大白话操控 AI，做出立等可用的程序。

为了方便毫无编程经验的读者入手，我们将从制作网页程序入手。因为其它程序都需要特定的编译器/解释器来处理这些程序，还是需要一些额外的相关知识。但是网页程序，直接在浏览器中打开即可运行，无需任何额外对编程的理解。


## 1. 编程的本质是“表达逻辑”，而不是“写代码”

很多人对编程的恐惧，来自于对一堆英文符号（如 `import`、`const`、`function` 等）的陌生感。

但事实上，写代码只是“实现手段”，核心的“编程思维”其实是逻辑表达。 
* **传统编程**：人类在大脑中想好逻辑 $\rightarrow$ 亲手把逻辑翻译成计算机能懂的编程语言（写代码） $\rightarrow$ 计算机运行。
* **AI 时代编程**：人类在大脑中想好逻辑 $\rightarrow$ 用母语（自然语言）讲给 AI 听 $\rightarrow$ **AI 翻译成代码** $\rightarrow$ 计算机运行。

```mermaid
graph LR
    A[人类大脑] -->|自然语言| B(AI 助手)
    B -->|自动翻译| C[计算机代码]
    C -->|直接运行| D(专属软件/工具)
    style B fill:#bbf,stroke:#333,stroke-width:2px
    style C fill:#bfb,stroke:#333,stroke-width:2px
```

我们不再需要了解“数组”，“循环”等等这些编程概念。我们只需要知道：**想让这个工具做什么，第一步是什么，第二步是什么。** 就可以编程了。



## 2. 用普通对话工具写出你的第一个程序

### 最简单的开始
首先要选用一个可以 直接生成可交互网页，当场打开玩的 AI 工具。最常用的几个AI工具，ChatGPT，Claude， meta.ai, gemini 都可以。如果你的 AI 工具不支持生成交互网页，也不要紧，我们下一章会介绍如何把 AI 生成的程序在网页中打开，甚至发不出去给朋友看。

我们用一个小游戏作为第一个 AI 生成软件吧。我们只要对着 AI 说出下面的话：


> 我想做一个飞机射击游戏。
> 玩家飞机在下方可左右移动自动射击，敌机从上方随机生成，有小、中、大三种血量，击毁有爆炸特效，显示分数和生命
> 打开画布，运行生成的游戏。 

“打开画布”是要求 AI 打开一个预览对话框，直接运行刚刚生成的程序。有些 AI 工具，只写“打开画布”，还需要手动的再对话框的 + 号按钮里选择“画布”才行。
点击发送按钮，等待一会之后，你就应该可以看到一个游戏程序了。

### 逐步升级

不要试图一口气让 AI 做出一个完美的复杂系统。先让它做出一个“能用”的最基础版本（MVP，最小可行性产品），然后在此基础上不断添加新功能。

例如：想做一个“背单词打卡工具”，可以先让 AI 做一个“能随机显示单词和释义”的页面，等这个做好了，再对它说：“现在在这个页面上加一个‘记住了’的打卡按钮和进度条”。

对于刚才生成的小游戏，界面应该是非常粗糙的，应该不会太令人满意，不过不要紧，我们现在可以要求AI对它进行优化。比如，告诉 AI：“让我的飞机有个大翅膀，喷着红色的尾焰...”


### 反馈与迭代（Continuous Feedback Loop）

如果在运行过程中发现任何问题，或者想调整界面，千万别慌，**继续在对话框里找 AI 吐槽**：
* **报错了**：直接把报错信息（甚至截图）发给 AI：“我双击打开后，点击按钮没反应，这是怎么回事？请帮我修复。”
* **想改样式**：直接说：“我觉得这个按钮太小了，帮我把它改成蓝色，并且居中显示。”
* **增加功能**：直接说：“现在这个计算器只能算加减法，帮我再加上乘除法功能。”

## 4. 复杂的实战演练

前面几个游戏都太简陋了。下面我们分多步骤实现一个复杂的 3D 游戏。下面的步骤，笔者在 meta.ai 上进行了实测可以生成非常漂亮专业的 3D 游戏。读者在实践时，可以根据喜好，修改每个步骤的内容制作属于自己的游戏：


复制每一段当作一次对话发送，AI 会在上一次的制品上继续迭代。

### 0. 初始化项目
```
创建一个 3D 低多边形飞行射击游戏，单文件 HTML。
使用 Three.js 用 importmap 引入。
玩家飞机在屏幕中央偏下，可左右上下移动，自动向前飞行。
相机跟随在飞机后上方。
有基础的星空背景和地面网格。
支持键盘 WASD/方向键 和 手机虚拟摇杆。
先给我可玩的原型，包含开始界面和游戏结束重开。
```

### 1. 战斗系统
```
加入自动射击和手动蓄力射击。
子弹是发光的细长圆柱体，有拖尾。
敌机分 3 种：小飞机直冲、中飞机左右摆动、大飞机血厚速度慢。
敌机从前方程序化生成，带简单 AI 会朝玩家大致方向移动。
子弹命中和敌机爆炸用粒子系统实现。
加入分数和连击显示。
```

### 2. 玩家系统
```
玩家有生命值 3 格，被撞或被敌弹击中扣血。
加入翻滚无敌：按 Shift / 双击屏幕 可侧翻躲避，0.8 秒无敌，冷却 3 秒。
加入能量条，蓄力射击消耗能量，能量随时间恢复。
UI 用中文：分数、生命、能量、连击。
```

### 3. 程序化关卡与难度
```
做无尽模式，难度随时间和分数递增。
每 60 秒出现一次 Boss 波次，Boss 有 3 种攻击模式：散射、激光扫射、追踪弹。
地面和云层程序化生成，远景有视差滚动。
加入小行星带障碍物，需要躲避。
```

### 4. Roguelike 成长
```
敌机被击毁有 20% 概率掉落道具：
1. 散射炮 2. 护盾  3. 能量恢复  4. 僚机
道具持续 15 秒，可叠加。
每 5000 分出现一次三选一升级：射速+20%、移速+15%、最大生命+1。
升级选择用暂停弹窗实现。
```

### 5. 视听打磨
```
美术：低多边形 + 发光材质，色调赛博蓝紫。
加入引擎尾焰、子弹拖尾、爆炸冲击波。
用 Web Audio API 合成：射击 biu、爆炸 boom、拾取 ding、Boss 警报。
背景音乐用程序化循环低音。
加入镜头震动和慢动作击杀特效。
```

### 6. 手机适配与体验
```
加入左侧虚拟摇杆控制移动，右侧按钮控制翻滚和蓄力。
自动射击默认开启，可在设置关闭。
UI 自适应竖屏和横屏，字体加大。
加入暂停按钮和设置面板。
死亡后显示本局分数、最高分、击毁数、存活时间。
最高分用 localStorage 保存。
```



## 5. 零基础玩家的避坑指南

非程序员在用通用 AI 写程序时，掌握以下几个小窍门可以事半功倍：

* **善用“Tailwind CSS”这个魔法词**：在你的指令里加上 *“使用 Tailwind CSS 来做界面设计”*。这是一个现代的样式框架，AI 极其擅长用它来写出非常高档、美观、响应式的界面，能瞬间让你的小工具具备大厂产品的质感。
* **不要去挑战复杂的“后台数据库”**：对于刚入门的新手，尽量做“前端工具”（即不需要注册账号、不需要联网存数据的工具，如计算器、排版工具、倒计时、小游戏、本地备忘录）。这类工具最容易通过单个 HTML 文件搞定，成功率 100%。
* **如果需要存储数据，让 AI 使用“LocalStorage”**：如果你希望自己写的小清单在关闭浏览器后数据不丢失，可以对 AI 说：*“请使用 LocalStorage 将待办数据保存在我的浏览器本地”*。这样，你下次打开这个 HTML 文件时，之前输入的内容依然会在，而且完全不需要复杂的服务器。

在这个 AI 浪潮下，**“创造力”的溢价正在急剧上升，而“实现成本”正在迅速归零。**
