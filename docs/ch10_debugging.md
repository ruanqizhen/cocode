# 第十章 橡皮鸭调试与全场景测试

> **“调试不是为了让程序能动，而是为了理解它为什么不动；测试不是为了证明自己没错，而是为了发现错在哪里。”**

---

## 10.1 生产级 Bug 排查法：堆栈跟踪、完整日志与复现步骤的组合灌输

当生产环境突发高危 Bug、报警邮件塞满信箱时，手忙脚乱地向 AI 问“我的服务器崩了怎么办”是毫无用处的。

要让 AI 成为优秀的调试结对伙伴，你必须学会科学地向它**灌输三维调试线索**。

### 📊 经典三维调试法（Triple-Line-Of-Evidence）
1. **线索一：完整的堆栈跟踪（Full Stack Trace）**：
   * 不要只截图，把终端最底层的报错堆栈（包括相关的 Error Code, Trace ID）完整复制为文本投喂。
2. **线索二：上下文关键日志（Contextual Logs）**：
   * 提供异常发生前 10 秒、发生后 5 秒的服务器或控制台完整输出，包括任何 SQL 查询、API 调用载荷。
3. **线索三：人类梳理的复现步骤（Reproduction Steps）**：
   * “1. 用户登录系统；2. 点击购物车结算；3. 用户并发双击了‘提交’按钮。”

:::tip 调试 Prompt
“根据以上三个线索，请扮演资深调试专家，分析该报错是在哪一行代码（指出可能的文件及行数范围）由于什么边缘条件被触发的，并给出两套不同的修复方案。”
:::

## 10.2 虚实之间：清醒面对 AI 无法感知真实运行环境的局限性

我们必须时刻保持冷静的**技术批判性思维**：

> **AI 不是神，它运行在沙盒和云端，无法真实物理感知你的操作系统环境。**

### ⚠️ 经典盲区
* **网络延迟与时序竞态**：AI 可能会写出逻辑完美的接口，但完全忽视了局域网丢包、高并发竞态锁导致的死锁。
* **硬件与权限局限**：大模型不知道你的云服务器其实只有 1GB 内存，或是当前的 Docker 容器缺乏写临时文件的系统权限。
* **时区与宿主机配置**：大模型往往理所当然地认为系统时区是 UTC，导致了时区换算引发的各种奇特业务 Bug。

人类架构师必须站在“物理真实环境”的法官席上，替 AI 补齐它无法感知到的现实世界物理约束。

## 10.3 密织测试网：单元测试、集成测试、以及边界条件的异常路径覆盖

优秀的软件系统不是靠“保佑”不崩溃的，而是靠**高密度的自动化测试网络**兜底的。

### 🌟 全场景测试矩阵构建

| 测试层级 | 推荐 AI 分工 | 覆盖核心指标 | 示例指令 |
| :--- | :--- | :--- | :--- |
| **1. 单元测试 (Unit)** | AI 100% 自动生成单元用例 | 覆盖函数的纯逻辑、死循环边界、冷门参数分支 | “为以下纯函数编写 Jest 单元测试，要求覆盖空值、溢出超长字符串和特殊字符边界。” |
| **2. 集成测试 (Integration)** | 人类定义 Mock 边界，AI 填充用例 | 数据库事务回滚、中间件拦截、多服务链条调用 | “使用 Supertest，模拟数据库连接中断场景，测试我的用户创建 API 是否能正确回滚事务。” |
| **3. 异常路径覆盖 (Chaos)** | AI 进行“混沌工程”头脑风暴，输出漏洞测试 | 恶意并发请求、非法 JSON 注入、超限 Payload 攻击 | “你现在是一个黑客，请为这个 API 编写 5 个可能导致拒绝服务（DoS）或 SQL 注入的异常测试用例。” |

通过构建这层海陆空立体防线，大模型在未来进行的任何微小重构，只要触发了旧逻辑的警报，测试网就会瞬间亮起红灯，为您彻底掐死线上事故的萌芽。
