# 非程序员的第一个程序

> 自然语言，正在成为这个时代最新、最强大的编程语言。

过去几十年里，“会编程”一直是一种技术壁垒。要想编程你需要学习复杂的语法；理解各种抽象概念；配置令人头皮发麻的开发环境；还要和那些神秘莫测的报错信息搏斗到深夜。

但今天，大语言模型（LLM）的出现，第一次真正改变了这一切。人类自己的母语，正在成为最新的“编程语言”。我们不再需要学习传统编程语言那些枯燥的语法，晦涩的关键字，就可以像“导演”一样，指挥 AI 写程序。

这一章，我们就来演示，如何在没有任何编程基础的情况下，指挥 AI 编写程序。为了方便快速上手，我们先从最容易成功的方向开始：编写“网页程序”。

网页程序有一个巨大的优势：不需要安装任何开发环境。浏览器，就是它的运行器。只需要用浏览器打开一个写着网页程序的文件，它就能立刻运行。（网页程序通常使用 HTML + JavaScript 语言编写）


## 表达逻辑

> 未来最重要的能力之一，或许不再是“会不会写代码”，而是“能不能清晰表达自己的想法”。

编程，很多人首先想到“程序代码”。但事实上，代码从来不是编程的本质。代码只是人类和机器沟通时使用的一种“中间翻译层”。

真正的编程，本质上只有一件事：描述你想让计算机做什么。

传统编程的流程是这样的：

```mermaid

    A[人类逻辑] -->|亲自翻译| B[程序代码]
    B -->|编译运行| C(计算机)

    style B fill:#bfb,stroke:#333,stroke-width:2px

```

而在 AI 时代，编程的流程变成了这样：

```mermaid
graph LR
    A[人类逻辑] -->|自然语言| B(AI 助手)
    B -->|自动翻译| C[程序代码]
    C -->|直接运行| D(计算机)

    style B fill:#bbf,stroke:#333,stroke-width:2px
    style C fill:#bfb,stroke:#333,stroke-width:2px

```

AI 接管了把逻辑翻译成代码的这部分工作，咱们人类自己就不再需要先学习“数组”“循环”“对象”“继承”这些编程所需的抽象概念了。我们所需要做的就是用自己的语言，清楚地描述：
1. 我想做什么；
2. 第一步发生什么；
3. 第二步发生什么；
4. 用户点击之后会怎样...
5. ...



## 第一个程序

目前大部分主流 AI 工具都已经支持直接生成网页应用。市面上比较常见的 AI 工具包括 ChatGPT、Claude、Gemini、Meta AI、Kimi、DeepSeek 等等。读者可以根据自己的喜好和便利程度任选一个进行实践。不同 AI 工具的表现并不会完全相同，但对于我们采用的示例程序会非常类似。

这一章的所有示例都是在 [Meta AI](https://meta.ai/) 上进行测试的。之所以选择 Meta AI 是因为它没有使用次数限制，而其它主流的 AI 工具的免费版基本都有使用次数限制。

如果只是简单问一些生活问题，我们使用 AI 工具的次数不会太多，次数限制不是个大问题。但对于编程来说，我们通常需要来来回回与 AI 沟通，调整我们的要求，AI 使用频率会比日常生活使用高得多。


### 从一个小游戏开始

咱们从最简单、最有反馈感的小项目开始。做一个飞机射击游戏。

具体步骤非常简单。打开 AI 工具，输入下面的要求，然后点击发送键即可：

```text
我想做一个飞机射击游戏。请直接生成一个可以运行的单文件 HTML，并打开画布并运行。
```

发送之后，等待几十秒。我们就会亲眼看到一个能够真正运行的游戏了。

### 什么叫“打开画布”？

很多 AI 工具支持一种类似“实时预览”的模式。可以理解为：AI 一边写代码，一边直接把程序运行给我们看。

有些 AI 工具会自动打开这个功能，比如 Meta AI；有一些（比如 Gemini）需要在对话框旁边的加号里选择添加“画布”来开启这个功能。

不同平台名字可能不同，比如叫 “Canvas”、“Artifacts”、“预览”、“运行”，等，但本质一样。

### 添加详细需求

你可能会觉得 AI 生成的这个小游戏，跟你想要的有些不一样，那是因为我们最初给他的指令太模糊了。只说要一个飞机射击游戏，射击什么，怎么射击，都没说。AI 只能自己猜我们的想法，然后随便造一个。

我们把问题描述的更具体，AI 生成的游戏也就更接近我们想要的，比如我们可以这样说：

```text 
我想做一个飞机射击游戏。
玩家飞机在下方，可左右移动并自动射击。
敌机从上方随机生成。
有小、中、大三种敌机血量。
击毁后有爆炸特效。
显示分数和生命值。
请直接生成一个可以运行的单文件 HTML。
打开画布并运行。
```


## 一些特别适合新手的 AI 项目

除了上面这个小游戏，还有很多适合新手的项目，下面我们就列举一些成功率极高，容易获得成就感的小程序，供大家测试：

### 现代版贪吃蛇

```text
做一个现代版贪吃蛇。
地图 20x20。
蛇身有渐变色。
吃到食物后短暂加速。
支持穿墙模式切换。
支持手机滑动控制。
记录最高分。
请直接生成一个可以运行的单文件 HTML。
打开画布并运行。
```

### 2048 强化版

```text
创建一个 2048 游戏。
支持键盘方向键和手机滑动。
加入撤销一步功能。
合并时有粒子动画。
达到 2048 后可继续挑战 4096。
请直接生成一个可以运行的单文件 HTML。
打开画布并运行。
```

### 打砖块 + 真实物理

```text
做一个打砖块游戏。
挡板支持鼠标和手指拖动。
小球有真实反弹物理。
砖块有不同血量。
随机掉落多球和加长挡板道具。
请直接生成一个可以运行的单文件 HTML。
打开画布并运行。
```

### 像素风跑酷

```text
创建一个像素风无尽跑酷。
角色自动前进。
按空格或点击跳跃。
支持二段跳。
背景视差滚动。
死亡后显示分数与重开按钮。
请直接生成一个可以运行的单文件 HTML。
打开画布并运行。
```

### 古风文字冒险

这个特别适合历史爱好者。

```text
做一个三国题材文字冒险游戏。
玩家扮演谋士。
每个场景提供 2-3 个选项。
不同选择影响结局。
支持存档。
界面古风。
请直接生成一个可以运行的单文件 HTML。
打开画布并运行。
```
